# Transferable skills matching

> Adapted from [job-search-skills](https://github.com/sameergdogg/job-search-skills) matching philosophy. Match on transferable skills, not job titles alone.

## Core rule

**Read listing snippets before judging fit.** Titles are misleading — a "Platform Engineer" post may match a DevOps CV; a "Software Engineer" post may be backend-heavy.

## Transferable domains (examples)

| User domain | Often matches titles |
|-------------|---------------------|
| DevOps / SRE | Platform, Infrastructure, Cloud, Release, Build, Observability |
| Backend | Fullstack, API, Services, Distributed systems |
| Frontend | UI, Web, Design systems, Product engineer |
| Data | Analytics, ML platform, BI, Data platform |
| Security | AppSec, DevSecOps, Cloud security |
| PM / TPM | Program manager, Product owner, Delivery lead |

## Tiered matching

1. **Direct** — query title ↔ listing title alignment.
2. **Transferable** — skill overlap from snippet/description ≥ ~50% of stated must-haves.
3. **Stretch** — include only if fewer than 5 strong matches exist in verified listings.

## Primary specialty

When the user's query names a specialty (e.g. "DevOps", "fullstack"), rank listings in that specialty above adjacent ones at equal skill overlap.

## Suggestions output

Generate 3–5 **short search queries** the user can paste into the app — vary seniority, remote/on-site, and adjacent titles (Tier 2).
