# Інфраструктурне середовище розробки (Hackathon DevOps)

  

Ласкаво просимо до робочого простору нашого MVP стартапу **Scout: Job Searcher AI Assistant**. У цьому репозиторії розгорнуто декларативне гібридне GitOps-середовище на базі `k3d` (Kubernetes), `ArgoCD` та `FluxCD`.

  

---

  

## Архітектура контуру (High-Level Design)

  

Інфраструктура побудована за принципом повного розділення відповідальності (Separation of Concerns) та автоматизації:

*  **ArgoCD** виступає центральним ядром та візуальним пультом управління всієї платформи (відповідає за деплой системних додатків та базових інфраструктурних шарів платформи).

*  **Flux CD** інтегрований у кластер як виділений контролер автоматизації ШІ-компонентів. Він нативно відстежує Git-репозиторій і миттєво доставляє зміни логіки безпосередньо в рантайм.

*  **AgentGateway (LiteLLM)** розгорнутий у неймсейсі `scout-platform` для забезпечення контуру безпеки (фільтрація Prompt Injection, маскування PII та централізоване управління API-ключами).

  

---

  

## Структура Монорепозиторію

  

```text

├── app/

│ └── skills/ # Керується FluxCD: версіоновані AI-вміння (SKILL.md) та промпти

└── platform/ # Керується ArgoCD: інфраструктурні маніфести ядра

├── base/ # Базові компоненти (Gateway, Flux System, Core Apps, Sync-політики)

└── environments/

└── dev/ # Оверлей для локального середовища розробки (Kustomization)

```

  

## Як це працює (SDLC Флоу)

Для Платформи (platform/): Всі інфраструктурні зміни описуються декларативно. Застосування та синхронізація з кластером відбувається автоматично після мерджу у гілку develop.

  

Для AI-Агента (app/skills/): Будь-яка зміна системних промптів, моделей чи файлів SKILL.md у папці app/skills/ підхоплюється контролером FluxCD протягом 1–2 хвилин у фоновому режимі. Не потрібно вручну оновлювати маніфести чи перезапускати поди — інфраструктура синхронізується самостійно нативно з Git.

  

Швидкий запуск та Доступи для команди

* **1. Вхід у середовище**

Просто запустіть цей репозиторій у GitHub Codespaces. Завдяки конфігурації .devcontainer локальний кластер k3d та всі мікросервіси в неймсейсі scout-platform (scout-api, scout-web, agent-gateway) піднімуться самі за кілька хвилин.

  

* **2. Запуск пульту управління ArgoCD UI**

Оскільки середовище крутиться в хмарі, для надійного доступу до веб-інтерфейсу прокиньте безпечний локальний тунель. Виконайте в терміналі:

  

```Bash

kubectl port-forward svc/argocd-server -n argocd 8085:80 --address 0.0.0.0 &
```
(Символ & запустить процес у фоні, консоль залишиться вільною).

  

* **3. Вхід в інтерфейс:**

Перейдіть на вкладку Ports (Порти) у нижній панелі VS Code.

  

Знайдіть порт 8085. Переконайся, що в Port Protocol стоїть HTTP.

  

Натисніть на іконку глобуса (Open in Browser) або клікніть на синє посилання.

  

Логін: admin

  

Пароль: Кластер генерує унікальний пароль при кожному запуску середовища. Отримати його можна однією командою:

  

```Bash

kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d; echo
```
**🔗 Корисні Endpoints всередині кластера**

Внутрішній Endpoint безпекового шлюзу (LLM Proxy):

  

```Plaintext

[http://agent-gateway.scout-platform.svc.cluster.local:8080](http://agent-gateway.scout-platform.svc.cluster.local:8080)
```
Перевірка статусів синхронізації Skills (через термінал):

  

```Bash

kubectl get kustomizations.kustomize.toolkit.fluxcd.io -A

kubectl get gitrepositories -n flux-system
```
Перевірка роботи подів платформи:

  

```Bash

kubectl get pods -n scout-platform
```
