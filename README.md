## Середовище розробки та GitOps Інфраструктура (Dev Env)

Для реалізації інженерного контуру стартапу **Scout: Job Searcher AI Assistant** було спроектовано та розгорнуто декларативне гібридне GitOps-середовище на базі `k3d` (Kubernetes), `ArgoCD` та `FluxCD`. 

### Архітектура контуру (High-Level Design)
Інфраструктура побудована за принципом повного розділення відповідальності (Separation of Concerns) та GitOps-автоматизації:
* **ArgoCD** виступає центральним ядром та візуальним пультом управління всієї платформи (відповідає за деплой системних додатків та базових шарів).
* **Flux CD** інтегрований всередину ArgoCD як виділений контролер автоматизації ШІ-компонентів. Він нативно відстежує Git-репозиторій і миттєво доставляє зміни логіки безпосередньо в кластер.
* **AgentGateway (LiteLLM)** розгорнутий у неймсейсі `scout-platform` для забезпечення контуру безпеки (фільтрація Prompt Injection, маскування PII та централізоване управління API-ключами).

---

### Структура Монорепозиторію (Platform Part)
```text
├── app/
│   └── skills/               # Керується FluxCD: версіоновані AI-вміння (SKILL.md) та промпти
└── platform/                 # Керується ArgoCD: інфраструктурні маніфести
    ├── base/                 # Базові компоненти контуру (Gateway, Flux System, Sync-політики)
    └── environments/
        └── dev/              # Оверлей для локального середовища розробки (Kustomization)
```
## Як це працює (SDLC Флоу)
Для Платформи: Всі інфраструктурні зміни описуються у platform/. Застосування відбувається автоматично після мерджу у гілку develop.

Для AI-Агента: Будь-яка зміна системних промптів, моделей чи файлів SKILL.md у папці app/skills/ підхоплюється контролером FluxCD протягом 1-2 хвилин у фоновому режимі. 
Hе потрібно вручну оновлювати маніфести чи перезапускати поди — інфраструктура синхронізується самостійно нативно з Git.

## Корисні інструменти та доступи у Dev-середовищі
* Пульт управління ArgoCD UI (Додаток: scout-root):
``` URL
http://localhost:8080
```
* Внутрішній Endpoint безпекового шлюзу (LLM Proxy): 
``` URL
http://agent-gateway.scout-platform.svc.cluster.local:8080
```
* Перевірка статусів синхронізації Skills (через термінал):
``` bash
kubectl get kustomizations -n flux-system
kubectl get gitrepositories -n flux-system
```
